# TwoSigma — Платформа для A/B тестирования

🧪 Аналог платформы **TriSigma от Авито** для управления экспериментами и A/B тестирования.

## 📋 Описание

TwoSigma покрывает полный цикл A/B-тестирования:
- Создание и управление экспериментами
- Feature Flags для управления функциями
- Детерминированное сплитование пользователей
- Расчёт метрик и статистической значимости
- Автоматическая подгрузка данных из подключённых БД
- Интерактивные дашборды и визуализация

## 🚀 Возможности

### Управление экспериментами
- ✅ Создание/редактирование/удаление A/B тестов
- ✅ Запуск, пауза, остановка экспериментов
- ✅ Поддержка множественных тестовых групп
- ✅ Настройка долей трафика

### Feature Flags
- ✅ Создание и управление флагами
- ✅ Rollout по процентам
- ✅ Детерминированная проверка для пользователей

### Метрики и аналитика
- ✅ Конверсия, CTR, средний чек, retention
- ✅ Статистическая значимость (p-value, z-test)
- ✅ Доверительные интервалы
- ✅ Расчёт размера эффекта (Cohen's d)
- ✅ Анализ статистической мощности

### Подключение к данным
- ✅ SQLite (встроено)
- ✅ PostgreSQL
- ✅ MySQL
- ✅ Автоматическая подгрузка данных

### Визуализация
- ✅ Интерактивные графики (Plotly)
- ✅ Дашборды экспериментов
- ✅ Графики конверсии и lift
- ✅ Временные ряды

## 📦 Установка

### 1. Клонирование репозитория

```bash
cd D:\Python\TwoSigma
```

### 2. Создание виртуального окружения

```bash
python -m venv venv
venv\Scripts\activate  # Windows
# или
source venv/bin/activate  # Linux/Mac
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Настройка конфигурации

Скопируйте `.env.example` в `.env` и настройте параметры:

```bash
cp .env.example .env
```

Отредактируйте `.env`:

```env
# База данных
DB_TYPE=sqlite
DB_HOST=localhost
DB_PORT=5432
DB_NAME=twosigma
DB_USER=postgres
DB_PASSWORD=your_password

# Статистика
SIGNIFICANCE_LEVEL=0.05
MIN_SAMPLE_SIZE=100
```

## 🎯 Запуск

```bash
streamlit run app/main.py
```

Приложение откроется в браузере по адресу `http://localhost:8501`

## 📖 Использование

### 1. Создание эксперимента

1. Перейдите на страницу **"Новый эксперимент"**
2. Заполните название и описание
3. Настройте группы (контрольная и тестовая)
4. Выберите основную метрику
5. Нажмите **"Создать эксперимент"**

### 2. Запуск эксперимента

1. Перейдите на страницу **"Эксперименты"**
2. Выберите эксперимент
3. Нажмите **"▶️ Запустить"**

### 3. Анализ результатов

1. Перейдите на страницу **"Аналитика"**
2. Выберите эксперимент
3. Введите данные (или подключите БД)
4. Нажмите **"🔬 Рассчитать"**
5. При необходимости сохраните результаты

### 4. Feature Flags

1. Перейдите на страницу **"Feature Flags"**
2. Создайте новый флаг
3. Настройте rollout
4. Используйте в коде для проверки

## 🔌 API для разработчиков

### Проверка фича-флага (Python SDK)

```python
from app.ab_tests.splits import UserSplits

# Проверка флага
is_enabled = UserSplits.check_flag(
    user_id="user_123",
    flag_name="show_new_checkout",
    enabled=True,
    rollout_percentage=50.0
)

if is_enabled:
    # Показать новую версию
    ...
```

### Определение группы пользователя

```python
from app.ab_tests.splits import UserSplits
from app.ab_tests.models import Experiment

group = UserSplits.get_user_group(
    user_id="user_123",
    experiment=experiment
)

if group.name == "treatment":
    # Показать тестовую версию
    ...
```

## 📊 Статистические методы

- **Z-тест для пропорций** — сравнение конверсий
- **T-тест Стьюдента** — сравнение средних
- **U-тест Манна-Уитни** — непараметрический тест
- **Доверительные интервалы** — для пропорций
- **Cohen's d** — размер эффекта
- **Power analysis** — мощность теста
